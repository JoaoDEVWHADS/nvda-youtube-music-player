# Contributing

How to contribute to YouTube Music Player.

## Ways to Contribute

1. **Report bugs** - Open issues for problems you find
2. **Suggest features** - Share ideas for improvements
3. **Translate** - Add or improve translations
4. **Code** - Fix bugs or implement features
5. **Documentation** - Improve docs

## Development Setup

1. Fork the repository
2. Clone your fork:
```bash
git clone https://github.com/YOUR-USERNAME/nvda-youtube-music-player.git
```
3. Set up build environment (see [Prerequisites](02-prerequisites.md))
4. Create a feature branch:
```bash
git checkout -b feature/amazing-feature
```

## Code Guidelines

### Style
- Use 4 spaces for indentation
- Follow PEP 8
- Keep lines under 100 characters
- Use meaningful variable names

### Commit Messages
```
type: short description

Longer explanation if needed.

Fixes #123
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`

### Pull Request Process

1. Update documentation if needed
2. Test your changes thoroughly
3. Ensure build passes
4. Open PR with clear description
5. Wait for review

## Translation Guidelines

1. Use formal language
2. Keep accessibility in mind
3. Test all translated UI
4. Check for text overflow

See [Translations](07-translations.md) for details.

## Issue Guidelines

### Bug Report
- NVDA version
- Windows version
- Add-on version
- Steps to reproduce
- Expected vs actual behavior
- NVDA log excerpt

### Feature Request
- Clear description
- Use case
- Proposed solution (optional)

## Code of Conduct

- Be respectful
- Welcome newcomers
- Focus on constructive feedback
- Keep accessibility as priority
